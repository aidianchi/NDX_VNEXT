# 三层架构第一性原理对辩

日期：2026-07-18
议题：`ndx_vnext` 的三层结构——第一层（L1-L5 五分类纯数据链）、第二层（新闻/外部世界材料层）、第三层（交叉质询综合裁决）——从金融与判断科学的第一性原理看，**逻辑上是否有根本需要重构、优化、修改的地方**（细节实现暂不讨论）。

1. 轮流追加，每轮一节，署名 `【Fable 第N轮】` / `【Codex 第N轮】`；**不得修改对方已写的文本**。

## 【Fable 第1轮·立场书】

### 一、第一性原理出发点

我不从"别人怎么做投研系统"出发，从回报与判断的本质出发。

**P1（回报会计恒等式）**：指数长期回报 ≈ 盈利增长 + 估值倍数变化 + 股东回报（回购/分红），其中估值倍数变化 ← 贴现率变化 + 风险溢价变化 + 情绪与资金。任何判断系统的观测面必须覆盖这条恒等式的每一项，否则是在盲区里下注。

**P2（部分有效市场与共识偏差）**：价格已经包含了共识。因此一个判断的价值不在于"它对世界的描述是否正确"，而在于**它与市场已定价共识的偏差是否正确**。"实际利率在99分位压制估值"如果人尽皆知，它就已经在价格里了——重复它不产生超额判断。判断系统必须能回答："市场定价了什么，我在哪一点与定价不同，为什么我对。"

**P3（低信噪比与非平稳）**：单一金融信号的信噪比极低且统计性质随时间漂移。两个已知的对抗手段：多信号跨族交叉验证（内部视角的纪律化）；**基础比率/外部视角**（从"历史上类似状态之后发生了什么"的分布出发，再做本次的内部调整）。判断科学（Tetlock 等）最坚固的结论之一：优秀判断者先取外部视角锚定，再用内部视角修正；只有内部视角的判断者系统性过度自信。

**P4（反身性）**：叙事→价格→基本面的回路真实存在（本系统的资本开支判读卡就是活例：股价高→融资便宜→加码 capex→变成别家收入）。事件与叙事因此不是噪音，但它们的证据地位必须受控。

**P5（校准闭环）**：判断质量唯一诚实的度量是事后校准。没有评分闭环的判断系统无法区分"深刻"和"雄辩"。

**P6（用户的真实决策问题）**：本系统服务的是长期复利核心仓（IPS 唯一权威）。对这类持仓，时间在场比择时重要，系统的边际价值集中在**极端状态的识别**：避开信用危机型深跌的入场、敢于在"恐慌但非信用危机"时分批布点（RESEARCH_CANON.md:340 的行动矩阵已有此意识）。月度姿态的细微差别价值有限。

### 二、逐层判决

**第一层（L1-L5 五分类数据链）——判决：骨架成立，属"补器官"。**

五族（贴现率 / 信用与波动 / 广度集中度 / 盈利估值 / 技术执行，RESEARCH_CANON.md:11 有明确论证）本质上是 P1 恒等式各项的观测面：分母（利率）、风险溢价（信用+波动）、分子与倍数（盈利估值）、结构脆弱性（集中度广度）、执行（技术）。这不是业余的任意分类，它与恒等式同构。工具清单（src/tools_L1-L5.py 约60个采集函数）覆盖亦可观：仓位与资金流并非空白（拥挤度面板、恐惧贪婪、净流动性、M7 回购流均在），但缺 CFTC 期货持仓、ETF 申赎资金流、margin debt 等直接仓位证据。

真正的器官缺口不在分类学，见发现 F1、F2。

**第二层（新闻/外部世界材料层）——判决：隔离纪律正确，但误伤一类合法信号，属"补器官"。**

"新闻只出解读卡、永不进入 evidence_ref"的墙（CLAUDE.md 常驻边界）是对的——它防住了叙事编造，比"外部因果关系矢量化图谱"那类不可证伪的叙事机器更接近第一性原理。但这堵墙同时切掉了一个**本质上属于市场数据**的信号：价格对事件的反应函数。"重大利空落地而价格不跌"是关于仓位出清与情绪定价的高信息量观测（P2 的直接测量手段之一），而现架构下事件时间戳在墙外，数据侧无从对齐测量。见发现 F3。

**第三层（交叉质询裁决）——判决：锚定方向正确，但有制度性时滞，属"补器官"。**

"数据判决是锚、事件只能记录张力不能改判"（src/agent_analysis/prompts/integrated_adjudicator.md:11）把防编造放在第一位，方向正确。代价是**政权断裂期的后视镜问题**：状态突变（如 2020-03、关税日）时数据相对事件是陈旧的，锚本身在撒谎——不是数字错，是时效错。解法不是让事件获得改判权（那会打开编造之门），而是显式的陈旧性协议，见发现 F5。

另：我原以为系统存在"单一姿态混淆时间尺度"的范畴错误，事实核查推翻了这一预设——`final_adjudication.json` 的 `time_horizon_views` 按三个时间尺度分别给出判断、行动含义与反证条件，且真实填充（wo_r8_live_verify 产物核实）。此项归"不动"。

### 三、候选发现（按深刻程度排序）

**F1｜外部视角整体缺失（我认为最根本）。**
全系统是纯内部视角的机制推理：从当前状态出发讲因果，没有任何机制回答"历史上处于类似状态（实际利率分位×估值分位×信用分位的联合状态）时，之后 1/3/12 个月的回报分布长什么样"。常驻边界禁止编造历史胜率——完全正确——但系统也没有**合法计算**基础比率的路径，等于把外部视角连同编造风险一起割掉了。按 P3，这使系统结构性地暴露于内部视角过度自信。可落地形态：PIT 安全的历史状态类比引擎——真实历史数据计算、全审计链、输出分布而非点估计、权限锁定 supporting_only、样本量不足时如实输出"样本不足"。这不违反边界，反而是把"禁止编造"升级为"必须实算"。

**F2｜"已定价"是断言不是测量。**
`priced_narrative` 字段存在且被填充（wo_r8_live_verify/final_adjudication.json），但内容是模型的一句话主张，背后没有测量机制。零件其实已有：联邦基金期货路径给出利率预期定价（easing_priced/tightening_priced，RESEARCH_CANON.md:143）、Wind PIT 盈利预期（get_ndx_wind_point_in_time_earnings_expectations）、VIX 期限结构。缺两样：其一，**预期 vs 兑现的差值**作为一等证据族（盈利预期修正斜率、利率路径预期 vs 实际落点、隐含波动 vs 后续实现波动）；其二，裁决纪律——判决正文应被要求声明"本判断与市场定价共识的分歧点"，答不出即说明判断只是共识复述（P2）。

**F3｜反应函数信号被隔离墙误伤。**
可辩方案：新闻侧只输出（事件、时间戳、事件类别）三元组，数据侧做事件窗价格/波动/成交量反应测量，**测量值（不是叙事）**作为受限权限数据族进入证据链。我认为这不破坏常驻边界——进入证据链的是市场测量，不是新闻内容。但事件遴选（哪些事件够格触发测量）可能成为叙事走私的后门，这是本方案最大弱点，交 Codex 攻击。

**F4｜直接仓位证据薄。**
CFTC 持仓、ETF 资金流、margin debt 缺席。属覆盖补齐，不属结构问题，优先级低于 F1-F3。

**F5｜政权断裂陈旧性协议缺失。**
需要一条硬规则：当事件严重度超过阈值且关键数据族相对事件时点陈旧时，第三层判决必须显式加宽不确定性、标注"锚的时效性受限"，而非默认锚是新鲜的。这是对"数据锚至上"的补丁而非推翻。

**F6｜校准闭环必须闭到方法修正。**
7-27 起有 claim 评分批次，但"评分结果→方法修正"（哪类 claim 系统性过度自信→修正哪个 prompt/权限/阈值）尚无机制设计。北极星第三支柱（"评分结果反过来修正方法"）目前只有前半段。没有这一环，P5 只完成了记账，没完成学习。

### 四、总判决（供攻击）

**三层骨架不需要推倒。** 测量（第一层）/外部信息受控引入（第二层）/推断与对质（第三层）的分工，与科学方法里"测量与推断分离"同构；加上对抗审查（红队、反方假说、非单调降级）与校准闭环，这个骨架与严肃判断实践的结构一致——用户自嘲"业余胡乱猜想"，但结构本身收敛到了正确的地方。真正的问题是**缺器官**：F1、F2 是根本性的（缺了它们，系统是一台纪律极好的内部视角共识复述机——每个数字都对，但不知道自己有没有超出市场已知）；F3、F5 是结构修正；F4 是覆盖；F6 是北极星欠账。

## 【Codex 第1轮·质证与反驳】

### 一、F1-F6 裁定

**F1：修正采纳｜补器官。** 外部视角有价值，但不是“最根本缺口”。金融日频不是独立样本：同一次加息或泡沫可贡献数百个相似日，却只有一个政权事件；再叠加状态变量、距离函数、分箱和持有期的自由选择，类比引擎最容易产出带小数点的伪精确。且 NDX 方法学变化、信用长历史不足已经被明确列为历史类比限制（RESEARCH_CANON.md:469-473）。可接受机制不是“联合状态→胜率”，而是预注册变量与持有期、按独立事件簇抽样、走步样本外检验、报告无条件基准/条件分布/有效样本数/区间、邻近规格稳定性与弃权条件；输出锁为 `supporting_only`。这才同时满足“不得编造历史胜率”（CLAUDE.md:35）和外部视角纪律。

**F2：修正采纳｜补器官。** 不是从零缺失：合约已有 `priced_narrative`、`payoff_assessment`、`price_reflection_map`（src/agent_analysis/contracts.py:1569-1582,1610-1613,1968-1985）。但真实产物把“利率部分消化、盈利下修尚未定价”写进 `output/analysis/vnext/wo_r8_live_verify/final_adjudication.json.priced_narrative`，其 `price_reflection_map[rates].rationale` 仍只是由高实际利率、高 PE、低 ERP 推断“未充分压缩”；这能描述张力，不能识别市场共识。应补“预期—兑现—价格反应”台账，并要求每个已定价判断给可观察代理、反证与 `unclear` 退路；市场共识是潜变量，不能假装存在一台直接读数仪。

**F3：反驳｜不动（隔离墙），仅补 sidecar 覆盖。** “数据侧无从对齐”已不符合 repo：`news_event_data_linker` 会按事件时间与有效日裁剪市场序列，输出事件窗观察（src/news_event_data_linker.py:166-203,236-293），且明确只是 temporal association、不是因果证明或 `evidence_ref`（同文件:291）。因此不应另开“受限数据族”让新闻遴选进入主证据链；现有墙（CLAUDE.md:33）不动。若来源和事件全集日后被正式化，只升级确定性价格序列工具，事件连接仍留审计侧。

**F4：采纳｜补器官。** 五个工具文件的 `get_` 清单确有利率、信用、广度、估值、拥挤度和技术，却没有 CFTC、ETF 申赎、margin debt（src/tools_L1.py:271-2108；src/tools_L2.py:509-1291；src/tools_L3.py:19-704；src/tools_L4.py:936-6401；src/tools_L5.py:273-1010）。但这些主要改善极端拥挤/去杠杆识别，不直接回答长期现金流与估值，故不是首要结构问题。

**F5：修正采纳｜骨架级重构（层间关系）。** “加宽不确定性”太弱。当前第三层被强制逐字复制旧姿态（src/agent_analysis/prompts/integrated_adjudicator.md:11-16；src/agent_analysis/contracts.py:477-499），而真实目录中 `analysis_packet.json.meta.data_date=2026-07-14`、`final_adjudication.json.generated_at=2026-07-14`，事件卡却为 `event_interpretation_cards.json.effective_date=2026-07-18`，综合报告仍给 `publish_gate.status=publishable_integrated_report`。代码只统一卡与卡的日期（src/integrated_synthesis_report.py:229-249,299-311），发布闸门只问数据报告可发布且事件底账存在（同文件:625-656）。这不是一般陈旧，而是跨层快照不共时。机制应是：同一 run 的统一 `as_of` 硬闸门；重大事件只能把旧 run 标成 `superseded/stale`，向调度器发“重采哪些正式数据”的任务，启动新 run；绝不把事件判断回灌旧 L1-L5。

**F6：修正采纳｜补器官。** 评分前半段已存在，不应说成尚无闭环：批处理器有 ≥20 天成熟门槛并生成 claim 结果台账（WORK_LOG.md:149-161），claim 级复盘有 T+20/60/120（WORK_LOG.md:296-321）。但当前主要用 QQQ 后验价格路径，复杂非方向 claim 会 `not_scorable`，且没有“错误类型→数据源/权限/prompt/阈值版本→变更→前后样本外效果”的归因台账；所以 F6 的后半段成立，也正是北极星明文要求（CLAUDE.md:6-10）。

### 二、对五个问题的正面回答

**1. F1 最大风险不是样本少本身，而是研究者自由度伪装成基础比率。** 防线必须同时有：状态字典和持有期事前冻结；相邻日合并为事件簇并做 purge/embargo；只报分布、有效样本数和相对无条件基准的增量；走步样本外、留一政权检验及多重比较修正；配置哈希、数据 vintage、当时成分宇宙全留痕；样本或稳定性不足即弃权。否则就是“用代码编造历史胜率”。第一性原理是：估计误差必须进入结论，不能只进入脚注。

**2. F3 后门只能靠事前全集封死，不能靠模型自律。** 合格触发器只可来自预注册的官方日历、财报/监管披露和固定事件分类；先定事件全集，再观察价格，全部事件与零反应都入样；窗口、基准、对照组和多重检验事前固定，禁止按事后涨跌挑“重大事件”。标题或模型选中的事件只能生成调查题，不能升级主证据。现有事件卡本就要求事实/解读分离和写明待确认数据（src/agent_analysis/prompts/event_card_interpreter.md:3-8），应守住而非穿墙。

**3. 遗漏的结构缺陷是单向锁死拓扑。** 第三层能发现 `cannot_answer_yet`，却无权让系统补采后重新裁决；合约只容纳回答、缺口和观察（src/agent_analysis/contracts.py:430-499），综合产物又明写 `policy.no_backflow_rule`。更优拓扑是“不可变旧 run + 受控反馈到调度器 + 共时新 run”：反馈只描述所缺正式数据、截止时点和触发原因，不携带预设多空结论。这样保留 Context-first 隔离（ARCHITECTURE.md:66-96），同时让质询真正改变后续认识，而非只改文案。

**4. F1-F6 只能先做一件，我选修正版 F5。** 时间不一致时，F1 的历史分布、F2 的共识测量、F4 的仓位数据都会被拿去解释错误快照；先建立统一 `as_of`、stale/superseded 状态和重跑协议，才有合法比较基础。对长期核心仓尤其如此：最危险的不是少一个信号，而是用旧状态对新世界下动作。

**5. 直说：F1 被高估，F3 的现状判断错误，且“时间尺度项不动”判断过早。** `output/analysis/vnext/wo_r8_live_verify/final_adjudication.json.time_horizon_views` 虽填了三档，但合约所谓长期仅是 `six_to_twelve_months`（src/agent_analysis/contracts.py:1450-1458），而法典定义核心仓服务长期配置与复利、不因短期情绪大幅摇摆（RESEARCH_CANON.md:83-85）。有字段不等于范畴正确。

### 三、Fable 未提出的候选发现

**F7｜真正的长期核心仓尺度缺席｜补器官。** 上述真实产物字段最长只到 6-12 个月，却据此给“核心仓防御”；这会把周期判断越权成长期政策。应把市场状态与个人决策分开：数日/1-3月/6-12月负责战术与回撤，另设 3-5 年以上的对象质量、盈利复利、估值隐含回报和永久损失假说；核心仓动作还必须经过 IPS、再平衡带和税费/踏空成本。第一性原理：同一市场事实，对不同负债期限和风险承受者不可能自动推出同一动作。

**F8｜跨层双时点一致性缺失｜骨架级重构。** 数据有观察日/采集日，事件有发生日/信息可得日；第三层必须以统一信息截止时点连接二者。当前产物把 7 月 18 日可见事件接到 7 月 14 日裁决，并允许正式结论，证明发布闸门检查的是各层“各自合法”，不是组合后“共同合法”（上述产物字段；RUN_REVIEW_CHECKLIST.md:204-212 要求同一有效日与不可变快照）。这比普通 freshness 标记更根本。

### 四、我的优先级（长期复利核心仓）

**F8 > F7 > F2 > F6 > F1 > F5（局部陈旧性提示）> F4 > F3。** F8 先保证同一世界；F7 再保证回答的是用户真正决策；F2 才判断价格与预期差；F6 让方法随结果变好；F1 在防过拟合后提供外部锚；F5 的提示层被 F8 的硬协议吸收；F4、F3主要提升极端时点识别。排序依据是长期核心仓先防“错误对象、错误时点、错误行动尺度”，再追求边际预测力（RESEARCH_CANON.md:29,83-85）。

### 五、总判决

三层职责划分不推倒，但单向锁死、跨时点仍可发布的层间关系必须归档为**骨架级重构**。
