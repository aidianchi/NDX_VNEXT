## Q1. 信用利差单位失真修复（优先级最高：当前报告正在误导读者）

**病根（已定位）**：`src/tools_L1.py` 的 `get_hy_oas_bp` / `get_ig_oas_bp`（以及同族 `get_hy_quality_spread_bp`）从 FRED 拉取 `BAMLH0A0HYM2` / `BAMLC0A0CM`，FRED 原始单位是**百分比**（如 2.71），代码原样透传数值，却在 payload 里声明 `"unit": "basis points"`。L2 分析员照标签写出"2.71 bp"，与真实值差两个数量级（实为 ≈271bp）。

**两条修法，推荐 B**：
- 方案 A：采集侧乘 100 换算成真 bp。名实相符，但历史快照里存的是百分比数值，新旧 run 对照、recompute 第二本账、模型历史写过的绝对阈值（如"收窄至 5bp 以下"）全部需要兼容处理，爆炸半径大。
- 方案 B（推荐）：采集侧把 `unit` 改为如实的 `"percent"`（函数名保留 `_bp` 不改——function_id 是跨 artifact 合约键，改名成本高，用 notes 注明命名历史）；L2 提示词与指标法典同步"该指标单位为百分比，行文引用须写 X.XX%（≈XXXbp）"；报告/抽屉展示层按 unit 字段渲染。数值零改动，历史快照零污染。
- 两案共同项：新增单元测试锁定 unit 声明与 FRED 原始单位一致；`RESEARCH_CANON` 对应判读卡补单位说明；下一次 live run 验收 L2 行文单位正确。
- 涉及：`tools_L1.py`、L2 提示词、canon、若干测试。**Codex 可施工，Fable 验收。**
