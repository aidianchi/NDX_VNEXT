## 当前稳定的数据

这些数据在本轮运行中足以支持对应层级的基本判断：

- L1：铜金比、净流动性动量等宏观流动性指标。
- L2：高收益债利差、投资级债利差等信用压力指标。
- L4 实时模式：成分股聚合 PE / Forward PE / FCF yield、NDX 简式收益差距、Damodaran 美国 implied ERP 月度参考锚、WorldPERatio / Trendonify / 蛋卷等当前页面交叉校验。
- L5：均线位置、MACD、OBV、RSI、Donchian、VWAP、MFI、CMF 等技术与资金流指标。

使用边界：

- 技术指标只能说明交易节奏和拥挤度，不能证明“便宜”。
- 代理指标只能说明方向或压力，不能当成官方事实。
- 估值指标不能自动推出看空结论，必须和盈利、利率、流动性一起看。
- 当前网页只能说明“当前页面怎么写”，不能自动说明“历史回测日当时能看到什么”。

## 当前可用但薄弱的数据

L3 是最重要的薄弱点。

本轮 L3 明确标出的弱项：

- `get_advance_decline_line`
- `get_percent_above_ma`
- `get_new_highs_lows`
- `get_mcclellan_oscillator_nasdaq_or_nyse`

这几类指标回答的是：

- 指数上涨是不是由很多股票一起推动；
- 还是主要靠少数头部公司拉动；
- 龙头强势有没有扩散；
- 广度恶化是否已经开始拖累趋势质量。

为什么重要：

> NDX 是市值加权指数。少数大公司很强时，指数可以很好看，但内部结构可能已经变窄。没有足够 L3 数据，系统就容易把“头部公司强”误读成“整个指数健康”。

---

## 当前不应做的事

- 不要把 L3 直接升级成 hard fail。它重要，但过早一票否决会让系统脆弱。
- 不要因为原始采集函数返回成功，就认为数据质量已经足够。
- 不要让 L5 技术趋势代替 L3 广度判断。
- 不要用没有证据的跌幅、点位、估值倍数或盈利阈值包装风险判断。
- 不要让联网搜索、bb browser 或网页爬取结果直接替代正式历史数据源。它们可以发现线索，但回测里默认只是待审候选。

## 历史数据缺口的补法

推荐新增一个独立的“历史数据研究助理”能力，而不是让主分析链临场补数据。

它的职责：

- 读取 `backtest_data_boundaries` 和 DataIntegrity notes，列出本次回测缺了哪些关键数据。
- 联网寻找候选来源，包括官网、SEC、Nasdaq、基金页面、存档网页、新闻稿、PDF、历史表格和数据供应商说明。
- 输出候选证据包：链接、发布时间、数据日期、摘录、截图或缓存路径、为什么可能适用、为什么可能不适用、置信度。
- 把每次探索过程也记录下来：怎么搜、哪个入口有效、哪个字段证明日期、失败在哪里、下次怎么验证。

重要边界：

- 候选证据包先标记为 `research_candidate` 或 `manual_review_required`。
- 未经人工确认或规则确认，不得直接进入 L1-L5 主证据。
- 如果同一方法在多次日期、多次来源上稳定可重复，才考虑升级为正式采集器或新的硬规则。

这条路线的价值在于：AI 不只是“一次性帮忙搜索”，而是慢慢积累“历史数据从哪里找、怎么验证、什么时候不能用”的经验，最终把探索沉淀成系统规则。

## 采集机 / 快照模式

当前网络现实是：yfinance/Yahoo 往往需要 VPN 出口更稳，DeepSeek API 又可能直连更稳。因此数据采集和模型推理应该允许拆开。

推荐运行方式：

1. 采集阶段只跑 `collect-only`，生成数据快照、`chart_time_series.json`、新闻 sidecar、数据边界和校验摘要。
2. 分析阶段只消费这个快照，运行 DeepSeek 和报告生成。
3. 快照视为不可变输入，报告里能追到它的采集时间、有效日期、跳过项和缺口。

部署选择：

- 同机分流：Yahoo/yfinance 走 VPN，DeepSeek 直连。优雅，但依赖规则配置稳定。
- 双机采集：闲置笔记本开 VPN 专做采集，主电脑直连跑 DeepSeek。朴素但稳。
- 不推荐：同一个完整 run 中手动来回切 VPN，容易产生半截数据、半截分析，难复现。

## 下一步数据工作

优先级从高到低：

1. 为 L3 广度指标寻找更稳定的数据源或 fallback。
2. 继续解决 Trendonify 或等价历史估值分位来源；不能静默退回 yfinance 当前值冒充历史分位。
3. 区分“采集成功”和“语义质量足够”，把质量边界写进数据层。
4. 扩展 L4 的盈利预期、盈利质量和估值压力数据；继续观察第三方估值源与成分股模型的 source disagreement。
5. 保持 L1、L2、L5 的现有稳定性，避免为了扩指标引入不可靠源。
6. 原型化历史数据研究助理，把联网探索沉淀成可复用的候选证据包和未来采集规则。
7. 产品化采集机 / 快照模式，让数据采集和 LLM 推理解耦。

完成标准：

- L3 能稳定说明广度、集中度、领导力扩散和内部动量。
- 数据缺失时，系统能明确降低置信度，而不是写成漂亮但空泛的判断。
- 每个关键数据源都有来源、频率、fallback 和可信边界说明。
